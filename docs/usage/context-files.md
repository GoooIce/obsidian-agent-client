# 上下文文件

使用上下文文件为你的 AI agent 提供额外的上下文。

## 什么是上下文文件？

上下文文件是特殊的 Markdown 文件，包含 agent 应该知道的指令、偏好或信息。这些文件会在会话开始时被 agent 自动读取。

::: tip
上下文文件是 agent 功能，不是插件功能。有关详情请参阅 agent 的文档。
:::

## 支持的上下文文件

每个 agent 使用自己的上下文文件：

| Agent | 上下文文件 |
|-------|-----------|
| Claude Code | `CLAUDE.md` |
| Codex | `AGENTS.md` |
| Gemini CLI | `GEMINI.md` |

将上下文文件放在你的**库根目录**中，让 agent 自动读取它。

## 示例

```markdown
# 库上下文

## 概述
这个库包含我的个人笔记和研究。

## 偏好
- 使用简洁、清晰的风格写作
- 列表使用项目符号
- 首选 Markdown 格式

## 重要笔记
- `Templates/` - 笔记模板
- `Daily/` - 日记

## 约定
- 日期使用 YYYY-MM-DD 格式
- 为笔记标记相关主题
```

## 最佳实践

1. **保持专注**：只包含相关信息
2. **定期更新**：保持上下文与你的库同步
3. **具体明确**：提供具体的示例和偏好

## 避免事项

- 敏感信息（API 密钥、密码）
- 非常长的文件（agent 有上下文限制）
- 过时的信息
